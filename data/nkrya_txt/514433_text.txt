 06. 07. 2004
 Adobe Intelligent Document Platform - интегрированная платформа для корпоративных служб работы с документами
 Задача проекта Intelligent Document Platform - сделать PDF с расширенной поддержкой XML универсальным форматом обмена данными для корпоративных систем
 Компания Adobe Systems подготовила новую серверную платформу на базе Java, которая призвана помочь автоматизировать и ускорить передачу данных в пределах организации в виде документов формата PDF.
 Одновременно Adobe представила несколько новых, а также модернизированных программных продуктов, которые опираются на данную технологию.
 Все эти продукты входят составной частью в Intelligent Document Platform.
 Задача данного проекта - сделать PDF с расширенной поддержкой XML универсальным форматом обмена данными для корпоративных систем.
 С помощью продуктов, выпускаемых Adobe, организации могут автоматически обрабатывать данные, пересылаемые в формах PDF, что избавляет от необходимости преобразовывать их в формат, поддерживаемый имеющимися корпоративными системами.
 Кроме того, данные из этих систем могут отображаться в файлах PDF.
 Таким образом Adobe противопоставляет себя своему конкуренту, корпорации Microsoft, которая планирует предложить XML в качестве основного формата для Microsoft Office и InfoPath, а также вступает в соперничество с более мелкими производителями.
 У нас есть единственный путь связать все разнородные источники данных на предприятия ",- считает Сидней Слоан, менеджер группы по маркетингу компании Adobe.
 Конечные пользователи уже при взаимодействии с базовыми системами передают и получают информацию в виде форм и документов.
 Первые предложенные Adobe серверные продукты были автономными; очень часто в распоряжении компании они оказывались вследствие приобретения чужих решений, не рассчитанных на взаимодействие.
 Теперь компания переработала эти продукты таким образом, чтобы они соответствовали новой интегрированной архитектуре на базе J 2 EE и XML. В результате новые продукты Adobe лучше подходят к расширенной ИТ-инфраструктуре.
 Java API и протоколы Web-служб позволят организовать взаимодействие средств обработки документов Adobe, например, с системами управления отношениями с клиентами ( Customer Relationship Management, CRM) и системами планирования ресурсов предприятия ( Enterprise Resource Planning, ERP).
 Кроме того, Adobe адаптирует своё программное обеспечение для работы с продуктами SAP и IBM.
 Как подчеркнул ведущий аналитик IDC Джошуа Дал, рынок электронных форм только начинает развиваться, и в прошлом году Adobe добилась серьёзных успехов в этой сфере.
 Переход на единую платформу на базе Java, по его словам, поможет компании и дальше расширять свой бизнес.
 Это ещё один шаг, позволяющий Adobe стимулировать продажи серверного корпоративного программного обеспечения.
 Предлагая продукты в архитектуре, опирающейся на стандарты, и поддерживая работу серверов Java-приложений, компания адаптирует их к тому, что сейчас становится общепринятой инфраструктурой для корпоративного программного обеспечения ",- заявил Дал.
 Продукты Adobe, по словам представителей компании, предназначены как для крупных предприятий, так и для государственного сектора.
 К примеру, формы PDF активно используются в работе Налогового управления и Иммиграционной службы США.
 Среди продуктов, которые подготовила Adobe,- Document Security Server, Barcoded Paper Forms Solution и Designer 6. 0.
 Компания также планиует представить обновлённые версии Form Server и Reader Extensions Server.
 К концу года Adobe начнёт выпуск Form Manager и Policy Server.
 Document Security Server и Policy Server позволяют организациям контролировать доступ к информации, создавать " аудиторский след ", показывающий характер использования ресурсов, и сертифицировать подлинность документов, заверяя их цифровыми подписями.
 Form Server ( предназначен для внедрения в документы динамических форм), Form Manager ( используется для публикации форм и управления ими) и Reader Extensions Server ( позволяет разблокировать скрытые возможности в модуле просмотра документов Adobe Acrobat) стоят 35 тыс. долл. в расчёте на процессор.
 Document Security Server будет предлагаться по цене от 50 тыс. долл., а расценки на Policy Server пока не установлены.
